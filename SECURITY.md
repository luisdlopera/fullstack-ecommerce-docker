# Security Policy

## Supported Versions

The latest commit on the main branch is supported for security fixes.

## Reporting a Vulnerability

Please do not create public issues for security vulnerabilities.

Use one of these channels:
- Open a private GitHub Security Advisory in this repository.
- Contact the maintainer directly and include reproducible steps, impact, and affected endpoints.

## What to Include

- Clear vulnerability description and impact.
- Reproduction steps or proof of concept.
- Suggested mitigation if available.
- Environment details (Node version, app mode, backend/frontend context).

## Response Targets

- Initial acknowledgement: within 72 hours.
- Triage update: within 7 days.
- Fix target for confirmed high/critical issues: as soon as possible, usually within 14 days.

## Disclosure

Please allow time for validation and coordinated remediation before public disclosure.
After a fix is released, a summary can be published with affected versions and remediation notes.
